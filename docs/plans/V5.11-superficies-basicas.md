# V5.11 — Superficies básicas (boundary / fill / thicken): CIERRA el Tier 2

## Contexto

Es el **último ítem bloqueante del Tier 2** del roadmap V5 ("superficies básicas
(boundary/fill/thicken)", hoy marcado "POR DEMANDA"). Cerrarlo completa el 100% de lo
bloqueante del flujo del vertical: lo que resta de Tier 3 (render Blender, PDM, plantillas
de plano por empresa) es cosmética real y queda por demanda. Tras esto, se retoma **V6.1
robustez** como siguiente ítem pro.

**Por qué es pro y no cosmética**: hoy Apolo solo hace sólidos y chapa desplegable (planos).
No puede modelar geometría de doble curvatura — **chutes, tolvas, deflectores y guardas
curvas** del vertical de transportadores — más que aproximándola. El combo real es
**superficie de contorno → darle espesor (thicken) → pared fabricable**. Eso llena un
hueco de capacidad genuino del kernel, no un adorno.

## Hallazgos de exploración (verificados en el código, no asumidos)

**El entorno YA tiene todo lo geométrico** (build123d 0.10.0 / OCP 7.8.1). Nada de kernel
nuevo de fondo, solo cablearlo:
- `Face.make_surface(exterior, surface_points=None, interior_wires=None)` → parche N-lados
  (BRepOffsetAPI_MakeFilling) → **Face**. Es la base de *boundary* Y *fill*.
- `Face.make_surface_patch(edge_face_constraints=…, edge_constraints=…, point_constraints=…)`
  → parche con **continuidad G1** a las caras de apoyo (la parte pro de *fill*).
- `thicken(to_thicken, amount, both=False, normal_override=None)` (módulo) y
  `Solid.thicken(surface, depth, normal_override)` → **Sólido** desde Face/Shell.
- `make_loft` YA existe (sweep.py:71) — no se toca.
- ⚠️ VERIFICAR las firmas exactas leyendo `.venv/Lib/site-packages/build123d/topology/two_d.py`
  y `operations_part.py` ANTES de usarlas (lección de la casa: leer la fuente instalada,
  no adivinar). `SetOffsetOnFace` sigue NO-GO en OCP 7.8.1 (no se usa aquí).

**Patrón de comando geométrico confirmado** (espejar exacto):
- Executor de creación: `def _exec_x(scene, cmd_id, p) -> None:` → construye base →
  `shape = place(base, p.position.tuple(), p.rotation.tuple())` →
  `scene[cmd_id] = Feature(cmd_id, p.name, shape, cmd_id)`.
- Executor de modificación (mira `_exec_fillet` registry.py:435, `_exec_shell` :474):
  `feat = _require(scene, p.feature)` · `edges = _resolve_sel(feat.shape, p.edges, "edge")`
  · `feat.shape = result` · `feat.make_unique()`.
- Errores accionables estilo Fix H (mensaje con el tope real, no el crudo C++ de OCCT).
- `path_from_points(points, smooth, closed)` (sweep.py:16) devuelve un **Wire**
  (Polyline/Spline); cierra el lazo si el primer punto ≈ último. Reutilizar para las curvas.
- Expresiones `=expr` por componente: se resuelven ARRIBA (validate_params/resolve_params),
  ANTES del executor — igual que `SketchSweepParams.path`. El executor NO lleva
  `wants_variables`.

**Suposiciones de sólido que una superficie (Face/Shell, volumen 0) rompe** — inventario:
- ✅ Serialización, malla/render, export STEP/STL: **funcionan solos** (tessellate y export
  son agnósticos a la topología; el log de comandos regenera, no se guarda geometría).
- ✅ Interferencias: una superficie no interfiere (volumen 0 < umbral) — correcto.
- ✅ cutlist/sheetset: ya tienen fallback `.solids() or [shape]`.
- ⚠️ BOM/masa/costeo: una Face da masa 0 / peso `None` → dejaría filas basura de peso 0.
- ⚠️ Proyección de secciones (`drawing/projection.py:174`): `if not half.solids(): raise` →
  una escena SOLO de superficies tumba la vista de sección.
- ⚠️ FEA (`fea/mesher.py`): exige volumen 3D; una superficie falla — su mensaje debe decir
  "dale espesor con thicken".
- ✅ drop_test: ya salta volumen ≤ 0 (correcto: una superficie no es cuerpo rígido).

**Decisión de ingeniería (criterio pro)**: una superficie **desnuda** (Face/Shell sin
sólidos) es **geometría de construcción/referencia** — no se fabrica ni se compra hasta que
se le da espesor. Por tanto se **excluye de BOM/masa/costeo** y se **filtra de la sección**.
Solo el resultado de `thicken` (un sólido) entra al mundo fabricable. Esto se centraliza en
un helper `is_surface(shape)` reutilizable (mandato de escala: frontera limpia).

## Comandos nuevos (3 — categoría "superficies")

### 1. `boundary_surface` — superficie desde curvas de contorno libres
Params (`models.py`): `name`, `curves: list[CurveSpec]` (cada `CurveSpec` = `{points:
list[list[float]], smooth: bool=False}`, `points` acepta `=expr` por componente como
`SketchSweepParams.path`), `points: list[list[float]] | None` (puntos 3D que la superficie
toca — modela un parche NO plano), `holes: list[list[CurveSpec]] | None` (lazos interiores),
`position: Vec3`, `rotation: Rot3`. Validador: `curves` no vacío.

Executor (`registry.py`): por cada curva → `path_from_points(points, smooth, closed=False)`
→ recoger `.edges()` de todas → `Wire(edges)` (build123d ordena; si no conectan → CommandError
accionable "las curvas no forman un lazo cerrado: los extremos deben coincidir"). Caso 1
curva cerrada (primer≈último punto): usar su Wire directo. →
`Face.make_surface(exterior=wire, surface_points=[Vector(*p) for p in points], interior_wires=…)`
→ `place(...)` → `Feature`. Cubre **boundary** (y *fill* de curvas libres — mismo motor
BRepOffsetAPI_MakeFilling). El shape es una **Face** (volumen 0).

### 2. `fill_surface` — parche sobre aristas de un sólido existente (la parte pro)
Params: `feature: str` (id del sólido), `edges: EdgeSelector` (x-selector edge; el lazo de
contorno del hueco/región), `tangent: bool=False` (continuidad G1 a las caras de apoyo),
`name: str="Parche"`. Emite una **Feature NUEVA** (no muta el target — es un parche que
luego se puede thicken/coser).

Executor: `feat=_require(scene, p.feature)` · `edges=_resolve_sel(feat.shape, p.edges,"edge")`.
Si `tangent`: mapear cada arista→cara vecina con OCP y `Face.make_surface_patch(
edge_face_constraints=[(e, f), …])`; si no: `Face.make_surface(exterior=Wire(edges))`.
Receta OCP para la adyacencia (idioma de `kernel/direct.py`, VERIFICAR sufijos `_s`):
```python
from OCP.TopExp import TopExp
from OCP.TopAbs import TopAbs_EDGE, TopAbs_FACE
from OCP.TopTools import TopTools_IndexedDataMapOfShapeListOfShape
m = TopTools_IndexedDataMapOfShapeListOfShape()
TopExp.MapShapesAndAncestors_s(feat.shape.wrapped, TopAbs_EDGE, TopAbs_FACE, m)
# por cada edge.wrapped → m.FindFromKey(edge.wrapped) da la lista de caras vecinas
```
CommandError accionable si las aristas no acotan una región (OCCT falla) o `make_surface_patch`
no está disponible con esa firma (fallback a make_surface sin tangencia + aviso). Emite
`scene[cmd_id] = Feature(cmd_id, p.name, patch_face, cmd_id)`.

### 3. `thicken` — superficie → sólido (pared)
Params: `feature: str` (una **superficie**), `thickness: float` (gt=0, `=expr`), `both:
bool=False` (simétrico ±t/2), `flip: bool=False` (invierte la normal). Muta EN SITIO (como
fillet/shell): la Feature-superficie se vuelve sólido.

Executor: `feat=_require(scene, p.feature)`; validar `is_surface(feat.shape)` → si no,
CommandError "thicken necesita una superficie (usa boundary_surface / fill_surface); '{name}'
ya es un sólido". Elegir la Face/Shell del shape →
`solid = thicken(surface, amount=(-p.thickness if p.flip else p.thickness), both=p.both)`
(módulo; fallback `Solid.thicken`). Validar `solid.volume > 0` (si no → CommandError
"el engrosado produjo un sólido vacío: revisa espesor/normal"). `feat.shape=solid;
feat.make_unique()`.

## Plomería de "superficie = construcción" (helper único)

`is_surface(shape) -> bool` en `kernel/shapes.py` (junto a `place`): `True` si el shape tiene
caras y **cero sólidos** (`not shape.solids()`). Usarlo en:
- **BOM** (`library/bom.py`), **masa** (`library/engineering/mass.py`), **costeo**
  (`library/costing.py`): saltar features con `is_surface` (no se cuentan como pieza).
  Preferible: filtrar en el punto donde iteran `scene.values()` (una guarda `if
  is_surface(f.shape): continue`), documentado.
- **Sección** (`drawing/projection.py`): construir el compound de corte solo con sólidos
  (`[s for f in feats if not is_surface(f.shape)]` antes del `&`) → una superficie ya no
  tumba la vista; el error existente "no interseca" solo si NO hay sólidos.
- **FEA** (`fea/mesher.py` o la capa API que arma el STEP): si el target `is_surface`,
  mensaje accionable "es una superficie; dale espesor con thicken antes del FEA".

## Tests (`tests/test_surfaces.py`, ~18-22) — patrón `d=Document(); d.execute(type, params)`

- **boundary**: 4 líneas en cuadrado → `d.scene[cid].shape`: `area≈lado²`, `volume≈0`,
  `len(faces())==1`, `len(solids())==0`, `tessellate()` no vacío, `bounding_box()` válido.
- **boundary no plano**: cuadrado + `points` con un punto elevado al centro → `area >` el
  plano; sigue siendo 1 Face.
- **boundary abierto**: curvas que no cierran → `CommandError` con mensaje del lazo.
- **boundary con hueco**: contorno + 1 `holes` → `area ≈ exterior − hueco`.
- **fill**: caja; seleccionar las 4 aristas de la cara `tope` → parche que cubre el tope
  (área ≈ cara). `tangent=True` vs `False` ambos producen Face válida.
- **fill hueco real**: caja con barreno pasante; seleccionar las aristas del barreno en la
  cara tope → parche que tapa el círculo (área ≈ π r²).
- **thicken**: boundary cuadrado (lado L) → `thicken thickness=3` → `volume ≈ L²·3`;
  `both=True` → mismo volumen, centrado en el plano; `len(solids())==1`.
- **thicken sobre no-superficie**: `thicken` de una caja → `CommandError` "necesita una
  superficie".
- **thicken paramétrico**: `thickness="=esp"` con `set_variable esp` → editar la variable
  regenera y cambia el volumen (candado de disciplina paramétrica).
- **construcción**: boundary_surface suelto → NO aparece en `get_bom`/`scene_mass_properties`
  (masa total sin la superficie); tras `thicken` SÍ aparece como sólido con masa.
- **sección robusta**: escena con boundary_surface + una caja → una vista de sección NO
  lanza; escena SOLO superficie → reporta "no hay sólido que cortar" (no crash).
- **export**: STEP/STL de una superficie no truena.
- **retro / candado**: suite completa sigue verde (917 → ~937); conteo de comandos 48→51.

## Riesgos → mitigaciones

| Riesgo | Mitigación |
|---|---|
| `make_surface_patch` firma distinta / no disponible en 0.10.0 | Fase 0 la verifica en la fuente; fallback a `make_surface` sin tangencia + aviso; no inventar |
| Adyacencia OCP `MapShapesAndAncestors` con sufijos/tipos equivocados | Copiar el idioma exacto de `kernel/direct.py`; micro-spike en Fase 0 |
| `Wire(edges)` no ordena curvas mal conectadas → error críptico | try/except → CommandError accionable con "extremos deben coincidir" |
| Excluir superficies de BOM rompe un test/consumidor existente | grep de consumidores de BOM/masa; los tests actuales usan sólidos → intactos |
| Filtro de sección deja fuera un caso legítimo | Solo excluye `is_surface` (Face/Shell sin sólidos); sólidos intactos byte-a-byte |
| thicken de doble curvatura auto-interseca | build123d usa Join=Intersection; validar `volume>0` y devolver error claro |
| Superficie desnuda confunde en el árbol UI | Documentar como geometría de construcción; (icono/etiqueta = follow-up UI, no bloqueante) |
